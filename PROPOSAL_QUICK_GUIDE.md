## ❓ COMMON QUESTIONS & ANSWERS (PROPOSAL)

### Q1: "Why this solution?"

**Answer:**
- "Existing solutions don't measure posture or sitting behavior"
- "We're proposing a chair-integrated solution that measures what wearables can't"
- "Physical interventions are more effective than notifications"
- "Privacy-preserving design addresses concerns"

### Q2: "Is this feasible in 2 months?"

**Answer:**
- "Yes - we've broken it down into manageable phases"
- "All components are readily available"
- "Using proven technologies (ESP32, Node.js, React)"
- "Systematic approach: hardware → firmware → backend → frontend"
- "We've allocated buffer time for integration"

### Q3: "What if sensors don't work as expected?"

**Answer:**
- "We've chosen proven sensors (FSR, MPU6050)"
- "Modular design - if one sensor fails, others continue"
- "Auto-calibration adapts to variations"
- "We'll test each component individually first"

### Q4: "How will you test this?"

**Answer:**
- "Unit testing for each component"
- "Integration testing at each phase"
- "End-to-end testing before demo"
- "User testing with volunteers"
- "Compare results with expected outcomes"

### Q5: "What's the patent angle?"

**Answer:**
- "Multi-modal sensor fusion algorithm - novel combination"
- "Adaptive intervention policy - context-aware"
- "Privacy-preserving work integration method"
- "Auto-calibration algorithm"
- "Sedentary risk index calculation"

### Q6: "What if you can't complete in time?"

**Answer:**
- "We've planned with buffer time"
- "Priority-based development - core features first"
- "Modular design allows incremental completion"
- "Minimum viable product approach"
- "Can demonstrate partial functionality"

### Q7: "How will you measure success?"

**Answer:**
- "Technical: All sensors working, real-time updates"
- "Functional: Posture detection accuracy >80%"
- "Project: Working prototype, successful demo"
- "Success criteria defined in slide 18"

### Q8: "What's the biggest challenge?"

**Answer:**
- "Sensor calibration for different body types"
- "Solution: Auto-calibration algorithm"
- "Real-time processing on ESP32"
- "Solution: Optimized algorithms, edge computing"
- "We've identified challenges and have solutions"

### Q9: "Why not use existing products?"

**Answer:**
- "Existing products don't measure posture"
- "No physical interventions"
- "Privacy concerns with camera-based solutions"
- "Our solution addresses these gaps"
- "Patent potential shows innovation"

### Q10: "What if components are unavailable?"

**Answer:**
- "Multiple suppliers identified"
- "Alternative components available"
- "Common components (ESP32, sensors widely available)"
- "Can substitute if needed"
- "Flexible design"
